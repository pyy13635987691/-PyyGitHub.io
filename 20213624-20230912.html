学习总结

简介:

Linux操作系统与Windo操作系统的目录结构不一样，例如：

1.在Windows操作系统种的结构为 D:\Git-2.30.0-64-bit\Git，且这种的结构可以有多个，就是结构树，可以有多个，

注意： 这里的D: 表示D盘,\表示层级关系

2.在Linux操作系统中的目录结构为 /Git-2.30.0-64-bit/Git ， Linux操作系统只有一个顶级目录，就是一个根目录

注意： 开头的/ 表示根目录,后面的/ 表示层级关系

学习Linux，本质就是学习在命令行下熟练使用Linux的各类命令：

1.命令行：就是Linux终端，是一命令行提示符页面。以纯“字符”的形式操作系统。

2.命令：就是Linux程序。一个命令就是一个Linux的程序。命令没有图形化页面，可以在命令行(终端)提供字符反馈。

命令的通用格式：commard [-options] [parameter]

commard:命令本身

-options:[可选，非必填]命令的一些选项，可以通过选项控制命令的行为细节

parameter:[可选，非必填]命令的参数，多数用于命令的指向目标

安装了GIT，一下绝大多数的命令是在书中看到（THE LINLIX COMMAND），并预习了前三章的内容，并熟习运行一些简单命令如：

编号 命令格式 命令作用 详解
1 cd [路径] 跳转或进入目录的命令
2 pwd 输出当前所在的工作目录
3 ls [-a -l -h] [Linux路径] 默认查看当前工作目录内的内容
4 mkdir [-p] Linux路径 在Linux路径下创建文件夹(相对路径与绝对路径都可以)
5 exit 退出GIT
6 cp [-r] 参数1 参数 2 可以用于复制文件夹
7 mv 参数1 参数 2 移动文件或文件夹
8 rm [-r -f] 参数1 ... 参数n 删除文件名为文件的文件
9 history 表示查看历史命令记录
10 rmdir 参数 删除目录（当目录内不为空时不能删除）
11 date 显示当前的时间和日期
12 help 查看一些命令
13 df 显示磁盘的内存占用情况
14 cat 参数 查看文件内的内容
15 more 参数 查看文件内的内容
16 git init 初始化git库
17 config --global user.name 设置用户名
18 config --global user.email 设置用户邮箱
19 touch 参数 创建文件名为参数的文件
20 git add 文件名(要带后缀) 将创建的文件或者是已有的文件从工作区转到暂存区
21 git commit [-m] 将暂存区的文件提交到库内，形成了一次记录
22 vi 参数 用vi编辑器进行,文件的编写
23 git status 查看文件的状态（包括库内的 暂存区内的 和未暂存区内的）
24 git log 查看日志信息
24 git reset --hard commitID 版本回退(回退到commitID所对应的版本)
